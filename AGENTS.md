# Investment Agent — Agent 指引

## 文档与变更（必读）

1. 阅读 `docs/GOVERNANCE.md` 与 `openspec/project.md`。
2. 实现功能前，确认存在活跃 change：`openspec/changes/<id>/`。
3. **契约真源**：`docs/`（requirements、api、data-model、workflow、frontend-contract 等）。
4. 按 change 内 **`tasks.md`** 执行；使用 `/opsx:apply` 或 Superpowers `executing-plans`。
5. **禁止**直接修改 L1 契约或新建 `docs/superpowers/plans/*.md` 作为规格真源。
6. 完成后 `/opsx:archive`，将 delta 合并回 `docs/`。

## OpenSpec 命令（Cursor）

安装 CLI 后在本仓库执行过 `openspec init --tools cursor`。在聊天中使用：

- `/opsx:propose <描述或 change-id>` — 新建变更包
- `/opsx:explore` — 探索代码库（brownfield）
- `/opsx:apply` — 按 tasks 实现
- `/opsx:archive` — 归档并合并规格

**安装**：需 Node 20.19+，`npm install -g @fission-ai/openspec@latest`，项目内 `openspec init --tools cursor`。修改 OpenSpec 版本后在本目录运行 `openspec update`。**重启 Cursor** 后 slash 命令生效。

## Propose 模板

- **手动指定**：`docs/templates/opsx-propose-prompt.md`（改 5 处占位符）
