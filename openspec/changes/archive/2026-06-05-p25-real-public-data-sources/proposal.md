# P25 真实公开数据源调研验证

## Why

P19/P20 已交付的是可配置只读 HTTP 数据桥接、ETF/基金证据 payload 解析、fixture/stub fallback 与信源分级能力；它们没有完成真实外部公开权威数据源的选型、接口验证、采集频率、字段清洗、合规边界和 smoke 验证。当前文档中“公开 HTTP 数据桥接”“A 股 ETF/基金证据源”的表述容易被误读为已经接通真实外部源。

下一轮若直接实现 collector，可能把公开网页入口误判为稳定 API，也可能误用需要授权的行情数据。因此 P25 先做调研验证和计划落地，确认哪些公开权威源可以安全、稳定、低频、只读接入，再将可实现部分拆到 P26/P27。

## What Changes

- 明确 P25 范围：真实公开数据源调研、接口验证、合规边界、字段标准化方案和阶段计划，不直接实现生产 collector。
- 修正 P19/P20 口径：已完成基础桥接与 parser；真实外部源 collector、90 天补采和 smoke 验证仍待后续阶段。
- 调研并验证候选源：巨潮资讯、上交所、深交所、证监会、基金业协会、东方财富基金、新浪财经、中证指数。
- 为每个源输出：权威性、可访问形态、可用字段、数据频率、限制/免责声明、失败策略、source_level/evidence_role 映射、入库路径。
- 排定后续阶段：P26 公告与证据 collector，P27 基金净值与 ETF 市场数据 collector。

## Out of Scope

- 不接券商交易 API。
- 不实现自动交易或账户自动变更。
- 不绕过登录、验证码、付费或授权限制。
- 不高频抓取行情或使用 Level2/授权行情。
- 不发送邮件、短信、系统 Push、Webhook 或 WebSocket。
- 不把第三方估算净值当作正式净值。
- 不在 P25 直接修改核心业务代码实现 collector。

## Acceptance

- `tasks.md` 中每个候选源都有调研验证结论或明确阻塞原因。
- 文档明确哪些源可先接入、哪些需二次验证、哪些暂不接入。
- P19/P20 当前能力与未完成真实源接入的边界在阶段文档中表达清楚。
- P26/P27 的候选范围、验收目标和安全边界清楚到可以继续写实现计划。
