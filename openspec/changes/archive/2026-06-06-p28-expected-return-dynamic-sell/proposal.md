# P28 预期收益与动态卖出评估增强

## Why

P0–P27 已完成本地纪律系统、证据链、复盘、首批真实公开公告/监管 collector，以及基金净值与 ETF/指数市场数据 collector。当前预期收益模块仍主要依赖样本数量生成静态占位情景，尚未完整体现 `docs/requirements.md` §9 对样本门槛、样本区间、筛选条件、情景收益、动态卖出评估和免责声明的要求。

P28 将在不预测未来、不承诺收益、不覆盖规则裁决的前提下，把预期收益模块升级为可解释、可降级、可审计的情景分析材料，并把当前净值与情景边界的关系转化为“卖出评估/重新评估”提示。

## What Changes

- 增强 expected return payload：补充 `sample_window`、`screening_condition`、`scenario_trigger`、`sell_evaluation`、`reassessment_trigger` 等可解释字段，并定义稳定 JSON/DTO schema。
- 强化样本门槛：样本数 `<5` 不生成收益区间；`5–19` 可展示区间但不展示精确概率；`>=20` 可展示概率。
- 引入动态卖出评估：当当前净值进入乐观情景下沿、突破基准情景上沿、跌破悲观情景下沿、基准情景中枢下移超过 15% 或达到用户目标收益时，只生成“评估/复核/记录人工计划”提示和可选动作，不自动交易。
- 保持规则边界：预期收益只作为分析材料，不覆盖 `final_verdict`，不替代 `RuleArbitrationNode`。
- 持久化与 API/前端契约对齐：decision detail 可复现样本数量、样本区间、筛选条件、情景和卖出评估提示。
- 补测试：覆盖样本门槛、概率展示、动态卖出提示、不覆盖最终裁决、持久化和 DTO 渲染。

## Out of Scope

- 不接券商交易 API，不自动下单，不新增买入/卖出执行能力。
- 不预测未来确定涨跌，不承诺收益。
- 不引入付费、登录、授权或 Level2 数据源。
- 不新增外部通知、Webhook、邮件、短信、系统 Push 或 WebSocket。
- 不重写完整回测引擎；P28 仅做本地规则化样本门槛和情景解释增强。
- 不把 LLM 输出作为最终裁决或收益保证。

## Acceptance

- `go test ./...` 通过。
- 前端相关测试或构建命令通过，以 `web/package.json` 实际脚本为准。
- `openspec validate --all --strict` 通过。
- 少于 5 个样本时，预期收益返回 `unavailable`，无收益区间且有定性原因。
- 5–19 个样本时，预期收益返回 `insufficient`，可展示区间但不返回精确概率。
- 20 个及以上样本时，预期收益返回 `available`，展示上行/基准/下行情景、概率、样本窗口和筛选条件。
- 当前净值触达卖出/重新评估条件时，只输出卖出评估提示或可选动作，不更新账户、不产生交易能力。
- 用户目标收益触发仅在本地持仓或设置中存在明确目标收益字段时生效；缺失时必须降级为“不适用”。
- 预期收益材料不得覆盖最终规则裁决；所有正式建议仍由规则裁决节点生成。
