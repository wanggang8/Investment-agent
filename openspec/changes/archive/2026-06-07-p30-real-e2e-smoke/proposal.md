# P30 真实环境 E2E / Playwright smoke 验收

## Why

P0–P29 已完成后，后端测试、前端测试、构建、OpenSpec 严格校验和公开源 smoke 已覆盖主要代码路径，但“真实功能可用”仍缺少一层跨进程、跨 UI 的本地端到端 smoke 验收。

近期复审发现，真实源、CLI、API 与前端展示之间的集成问题往往不会被单元测试单独暴露，例如 JSON 空数组契约、公开源空窗口语义、预期收益展示和本地临时产物清理。因此需要单独建立轻量、可重复、默认安全的 E2E smoke 层，验证本地 server 与 web 在只读边界内可协同工作。

## What Changes

- 新增 P30 E2E smoke 验收范围，覆盖本地 API + 前端关键只读路径。
- 使用本地可控数据、stub 或临时 SQLite，不依赖真实密钥、登录源、付费源、券商接口或自动交易能力。
- 将 Playwright 或浏览器自动化产生的临时产物纳入忽略/清理规则，避免 `.playwright-mcp/` 等日志目录污染工作树。
- 明确 smoke 只验证功能可达、关键 UI 不崩溃、降级态和审计可见，不替代全量业务测试或真实公开源专项 smoke。

## In Scope

- 本地启动 `cmd/server` 与 `web` dev/build 相关的 smoke 验收脚本或测试。
- 覆盖关键页面/路径：健康检查、仪表盘或首页、决策详情/预期收益展示、证据/审计或降级态展示。
- 使用临时配置和临时 SQLite，测试结束后清理。
- 更新文档和 OpenSpec 进度，说明 P30 的验收边界与命令。
- 保证新增测试不会写入真实密钥、不会访问券商交易 API、不会执行买卖操作。

## Out of Scope

- 不接券商交易 API，不实现自动交易，不发送买入、卖出、撤单、改单请求。
- 不引入登录、付费、授权、Level2 或用户身份行情源。
- 不引入邮件、短信、系统 Push、Webhook、WebSocket 或其他外部通知渠道。
- 不承诺收益，不预测确定涨跌，不让 DeepSeek 或任何模型覆盖规则裁决。
- 不把 Playwright smoke 扩展成高频抓取或绕过权限的浏览器爬虫。
- 不追补 P19–P24 历史 archive；如需治理追溯，应单独立项。

## Impact

- 提高本地交付前对真实 UI/API 协同问题的发现能力。
- 为后续阶段提供稳定 smoke 命令，降低“单元测试通过但功能不可用”的风险。
- 需要新增或调整少量测试脚本、配置示例、忽略规则和文档。
