# Design: P7 真实数据与分析底座

## Context

P0-P6 已完成本地工程、数据表、规则、工作流、API、前端基础页面和验收文档。P7 的目标不是改变产品边界，而是把数据和分析输入从占位能力提升为可替换、可降级、可审计的基础能力。

当前阶段按 `docs/development-plan.md` P7 执行，范围限定为：

- P7.1 真实行情与情报数据源。
- P7.2 RAG/VecLite 检索与索引。
- P7.3 DeepSeek 分析师材料。

## Goals / Non-Goals

**Goals:**

- 提供行情数据源适配层，支持真实数据源与本地 stub。
- 提供情报数据源适配层，写入 `intelligence_items`。
- 市场快照刷新写入 `market_snapshots` 与 `audit_events`。
- 提供 VecLite 索引读写、SQLite 重建和不可用降级。
- 将 `rag_chunks`、`intelligence_summary` 纳入检索构建。
- 提供 DeepSeek 客户端封装，并替换分析节点中的占位实现。
- DeepSeek 输出只作为分析材料，不写最终裁决。
- 外部依赖不可用时进入降级状态，并保留审计证据。
- 对非显然业务约束、降级路径、审计写入和禁止自动交易边界写中文注释。

**Non-Goals:**

- 不实现 P8 的前端图表、交互增强和前端测试。
- 不实现 P9 的 `cmd/agent`、周期复盘和本地交付说明。
- 不新增自动交易、一键交易或收益承诺能力。
- 不让 DeepSeek 替代规则引擎生成最终裁决。
- 不引入 P7 计划之外的数据供应商功能。

## Decisions

### Decision 1: 使用端口/适配器分离真实数据源与业务逻辑

行情和情报数据源通过接口进入应用层，具体供应商或 stub 放在基础设施层。业务层只依赖统一 DTO 和错误语义。

原因：P7 要支持真实数据源，也要在缺凭证或离线开发时使用 stub。这样可以让测试覆盖失败、过期和部分成功场景。

备选方案是直接在 workflow 中调用外部 API，但会让降级、审计和测试分散。

### Decision 2: 市场刷新必须先形成可审计结果

市场刷新应区分全部成功、部分失败、全部失败、数据过期和写入失败。成功或降级都需要写入审计事件；写入失败不得留下不一致快照。

原因：P6 的 A16 已定义市场刷新验收路径，P7 需要把真实数据路径接上，同时不破坏事务和审计要求。

### Decision 3: VecLite 是检索加速层，SQLite 是可恢复来源

`rag_chunks` 和 `intelligence_summary` 作为可恢复文本来源，VecLite 索引用于检索。索引不可用时使用 SQLite 摘要；摘要也不足时返回信息不足。

原因：VecLite 索引属于派生数据，SQLite 中的文本和摘要是本地可恢复基础。这样可满足 P7 的索引重建与降级要求。

### Decision 4: DeepSeek 只产生 analyst_reports

DeepSeek 客户端输出被解析为 `analyst_reports` 或等价结构，工作流随后交给规则引擎生成最终裁决。LLM 不可用、超时或输出不可解析时，工作流标记降级。

原因：项目边界要求 DeepSeek 不写最终裁决。这样既能利用分析材料，也能保持规则可解释、可审计。

### Decision 5: 配置默认支持本地开发

配置应包含数据源开关、stub 选项、VecLite 索引路径和 DeepSeek 环境变量名称。示例配置不得包含真实密钥。

原因：P7 引入外部依赖，默认本地环境仍应可测试、可启动。

## Risks / Trade-offs

- 外部数据供应商不可用 → 使用 stub 和降级路径，测试覆盖失败与过期场景。
- VecLite 索引损坏或缺失 → 从 SQLite 文本块重建；重建失败时返回信息不足。
- LLM 输出格式不稳定 → 使用结构化解析和失败降级，保留原始错误摘要到审计上下文。
- 真实数据和分析材料可能被误当作交易指令 → 代码注释和 DTO 命名强调分析材料属性；最终裁决仍由规则引擎生成。
- 引入外部依赖导致测试变慢 → 单元测试默认使用 stub，不依赖公网。

## Migration Plan

1. 增加配置字段和示例，默认使用本地 stub。
2. 增加行情、情报、VecLite、DeepSeek 接口与基础设施实现。
3. 在应用层串联市场刷新、检索和分析服务。
4. 补充降级、审计和事务测试。
5. 执行 P7 三组验收命令。

## Open Questions

无阻塞问题。具体真实数据供应商、请求频率和 DeepSeek prompt 细节在实现中以可替换接口承载，不扩大 P7 范围。
