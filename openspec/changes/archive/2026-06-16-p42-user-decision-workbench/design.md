# P42 User Decision Workbench Design

## Context

P33-P41 已补齐账户/持仓、风险预警、规则治理、真实 LLM、RAG/VecLite 检索质量、前端完整旅程、本地运维恢复和 P40 后路线图治理。P42 的目标不是新增投资决策引擎，而是把既有能力整理成用户每天可直接使用的工作台。

## Decision

P42 采用前端组合优先的实现方式：

1. 新增 `/workbench` 页面，并把主导航中的首页或新增入口指向“决策工作台”。
2. 页面并行读取既有 DTO：今日 dashboard、今日纪律报告、持仓/组合、风险预警、规则提案、复盘摘要、设置/运行状态。
3. 页面分为四个区域：
   - **今日先看**：今日纪律报告、最终裁决状态、缺前提或降级原因。
   - **组合与风险**：账户/持仓摘要、风险预警、禁止动作和建议人工复核入口。
   - **规则与复盘**：待确认规则提案、近期复盘摘要、审计/追踪入口。
   - **主动咨询入口**：进入现有决策咨询页，带安全说明；不在工作台自动触发咨询请求。
4. 所有入口只导航、过滤或展示本地事实；不自动提交确认、不自动应用规则、不触发交易、不外部推送。

## Boundaries

- 不新增券商 API、自动交易、外部推送、自动确认、自动规则应用或收益承诺。
- 不直接读取 SQLite、VecLite、本地日志或文件。
- 不把 LLM 输出作为最终裁决；工作台只展示已有分析材料和规则/守门人链路状态。
- 不新增登录源、付费源、授权源、Level2 或高频源。

## Verification

- Vitest 覆盖工作台组件/页面的成功、空库、降级、错误和安全文案。
- Playwright smoke 覆盖 `/workbench` 可达、核心区域可见、窄屏不遮挡和无自动交易/自动应用规则入口。
- OpenSpec 校验通过。
- 后端测试仅在新增或修改后端 DTO/handler 时运行；否则以前端测试、build 和 E2E 为主。
