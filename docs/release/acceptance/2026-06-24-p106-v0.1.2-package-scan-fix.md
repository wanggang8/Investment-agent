Verdict: `v0_1_2_release_package_scan_fix_passed`

## Trigger

After P105, commit `d46a7e9` and tag `v0.1.1` were pushed. GitHub Actions started:

- Release run `28073059003`: failed.
- CI run `28073058103`: failed.
- Security Scan run `28073058085`: passed.

Both failed workflows stopped at release package smoke/build. The failing log showed:

```text
forbidden content pattern in web/src/pages/DataQualityPage.tsx: (?:^|[,{]\s*)[\"']?prompt[\"']?\s*:\s*[\"'][^\"'\n]{16,}
```

## Root Cause

`web/src/pages/DataQualityPage.tsx` passed a caller-specific redaction label shaped like a JSON prompt payload:

```ts
prompt: 'prompt [REDACTED]'
```

The value was redacted, but the source shape still matched the release package scanner's strict prompt-payload pattern. The correct fix is to keep the scanner strict and use a short neutral replacement label.

## Fix

- Changed the Data Quality page redaction label to `prompt: '[PROMPT]'`.
- Bumped current source metadata from `v0.1.1` to `v0.1.2`.
- Added this P106 acceptance record and OpenSpec archive.

P106 does not move or delete the already pushed `v0.1.1` tag.
